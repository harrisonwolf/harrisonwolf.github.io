boij-soderberg-engine

C++ · github.com/harrisonwolf/boij-soderberg-engine · arXiv:2512.24320

We were using Macaulay2 to do the search initially, and after a little while, that's when I started to write my engine instead. Macaulay2 was painfully slow, and I could feel it in my bones that I could make something faster. Just as the core was getting finished, and essentially at the same time that my engine found the fourth or fifth counterexample — whichever it was — my advisor realized the pattern. We were able to rigorously prove it, discovering one of the paper's three main theorems (arXiv:2512.24320).

My engine fed the other side of the same result, too. The first search had been turning up examples of what became an infinite family of counterexamples — all following exactly Pell's equation (think of it like the Fibonacci sequence, but in algebra), which itself is, in no exaggerated way, a genuine miracle. The engine surfaced the counterexamples: once you bound their size, there are exactly four. The engine found them; the proofs are the paper's — pure number theory for the infinite family, and a by-hand case analysis, more than a dozen cases, for the four. It is a great illustration of how pure and abstract mathematics can be, while at the same time being so utterly unwieldy. Given the number-theoretic and commutative-algebra methods we have now, we were able to find beautiful, essentially miraculous connections and explanations for one of the results; the others remain more and more hidden — we simply lack the tools and the vision for them so far.

The objects themselves, in Macaulay2's BoijSoederberg package: degree sequences, the pure Betti table of one small sequence, and the Betti numbers of two of the large sequences from the research files — three-digit degrees already push the Betti numbers into the millions.

Benchmarks

Fresh task-matched runs compare the same candidate families, full pure Betti calculations, explicit BEH/LLBC checks, and gcd rinse. The evidence studio alternates execution order, records external wall time and peak RSS, and rejects a bundle unless the complete result arrays match exactly.

Across four headline cases in codimensions 3, 5, 6, and 7, the C++ engine ran 21.4–61.6× faster by median end-to-end wall time. The largest case contained 1,221,759 candidates and took 4.087 s in C++ versus 87.594 s in Macaulay2. All 12/12 paired runs succeeded and returned identical complete bad-sequence and gcd-rinsed arrays.

How the engine is built

The core represents the actual mathematical structures and runs the calculations; a recursive enumeration algorithm generates those structures inside it; and a performance layer runs the computations and checks them against the conjectures, within the hardware and runtime constraints and the inherent difficulty of large-integer arithmetic on a 64-bit processor.

And over both variables at once, on a linear scale, out to degree 150: codimension 3 has reached 551,300 candidates; codimension 7 is at 294,109,729,200. At a fixed codimension the count grows like dᶜ/c! — concretely, at codimension 3, raising the max degree from 100 to 1,000 takes 161,700 candidates to 166,167,000, about a thousand times more.

An abridged excerpt from the supported exact path, beginning at cross-cancellation and including the checked LCM loop:

pair<long long,long long> reduced_product_fraction(
    vector<long long> numerator_factors,
    vector<long long> denominator_factors) {
    // Cross-cancel before multiplying.
    for(long long& numerator_factor: numerator_factors){
        for(long long& denominator_factor: denominator_factors){
            const long long divisor = gcd(numerator_factor, denominator_factor);
            numerator_factor /= divisor;
            denominator_factor /= divisor;
        }
    }

    long long numerator = 1;
    for(long long factor: numerator_factors){
        numerator = checked_multiply_nonnegative(
            numerator, factor, "pure_betti reduced numerator/final value");
    }
    long long denominator = 1;
    for(long long factor: denominator_factors){
        denominator = checked_multiply_nonnegative(
            denominator, factor, "pure_betti reduced denominator/LCM");
    }
    return {numerator, denominator};
}

        L, p.second, "pure_betti denominator lcm");

// Clear each denominator before the final checked multiplication.
for(int i=0; i<c; i++){

Abridged from src/seq_funcs.cc at the audited commit. Each πi is cross-cancelled before checked numerator and denominator products; the checked LCM loop combines those reduced denominators; then the code divides each denominator out before the final checked multiplication. The search evaluates the complete exact Betti vector with explicit BEH and LLBC checks—there is no L-threshold auto-pass.

Ownership

The core is a ~1,200-line C++ engine I wrote by hand in 2024, cross-validated case-by-case against Macaulay2's own BoijSoederberg package. I later used an AI agent to reorganize the repo, write the docs, and add the test and benchmark harness. The engine found the counterexamples; the proofs are the paper's.
