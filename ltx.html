ltx-studio

Python · github.com/harrisonwolf/ltx-studio

It makes short videos from typed descriptions, and the whole job runs on an 8 GB laptop GPU. Every run keeps everything it records, so analysis harnesses I build later — on better hardware — can still read every past run. Whether an old run is still worth judging is a call I make case by case.

Setting up a run: the backend schematic (fast → nicer), a contextual INFO panel, and READOUT meters predicting VRAM, quality, drift, and render time before anything starts.
The LIVE tab mid-render. More screenshots and sample clips in the repo README.

The clip that exact run produced, beside a second render from the same studio:

Left: "Bioluminescent forest at night — glowing moths, luminescent moss, a stream lit from within. Cinematic, 35 mm." Right: fireflies over a glowing stream, 64 fps after interpolation.
How far it came in two weeks: the first clip this project ever produced (June 22, the initial LTX smoke test) beside a day-fourteen re-run of the same dog-on-a-beach prompt (July 6, Wan backend), same 8 GB card.

ltx-studio grew out of an earlier AnimateDiff pipeline I directed onto the same unsupported 8 GB card (a PyTorch-SDPA attention backport, a from-scratch LCM scheduler, a conv-aware LoRA merge) before modern open-weight models made the output worth watching. That earlier work, which I came to with no video-generation background, is where I learned the model internals this one runs on. Contact me for more info.

Full detail — architecture, the marker protocol, running it — in the README: github.com/harrisonwolf/ltx-studio
